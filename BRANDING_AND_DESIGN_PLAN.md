# QueryAI - Branding, SEO, Landing Page & UI Design Plan

## üìã Executive Summary

This document outlines a comprehensive strategy for QueryAI's branding, SEO optimization, landing page redesign, and UI/UX improvements. The goal is to create a cohesive, professional, and conversion-optimized experience that positions QueryAI as a **Fact Research Assistant** - a powerful tool for finding accurate, verified information quickly. While powered by AI technology, the focus is on research outcomes and factual accuracy, not the AI itself.

---

## üé® 1. BRANDING STRATEGY

### 1.1 Brand Identity

**Brand Name:** QueryAI
**Tagline Options:**
- Primary: "Your Fact Research Assistant"
- Secondary: "Research. Verify. Answer."
- Alternative: "Accurate Facts, Verified Sources"
- Short: "Research Made Simple"

**Brand Personality:**
- **Accurate** - Fact-focused, verified information
- **Trustworthy** - Reliable, source-cited research
- **Accessible** - Easy to use, no technical barriers
- **Efficient** - Fast research, quick answers
- **Professional** - Enterprise-ready, research-focused

**Positioning:** QueryAI is a Fact Research Assistant that helps users find accurate, verified information quickly. Powered by advanced technology (AI), it combines real-time web search with document analysis to deliver comprehensive, source-cited answers. The focus is on research outcomes and factual accuracy, not the underlying technology.

### 1.2 Color Palette

**Current State:** Orange-based (orange-50 to orange-100, orange-600)
**Recommended Evolution:**

#### Primary Colors
- **Primary Blue:** `#2563EB` (Blue-600) - Trust, intelligence, technology
- **Primary Blue Dark:** `#1E40AF` (Blue-700) - Depth, professionalism
- **Primary Blue Light:** `#3B82F6` (Blue-500) - Energy, innovation

#### Secondary Colors
- **Accent Orange:** `#F97316` (Orange-500) - Keep for CTAs, highlights
- **Accent Orange Light:** `#FB923C` (Orange-400) - Hover states
- **Success Green:** `#10B981` (Emerald-500) - Success states, positive actions
- **Warning Amber:** `#F59E0B` (Amber-500) - Warnings, attention

#### Neutral Colors
- **Gray Scale:**
  - `#0F172A` (Slate-900) - Primary text
  - `#1E293B` (Slate-800) - Secondary text
  - `#475569` (Slate-600) - Tertiary text
  - `#94A3B8` (Slate-400) - Placeholders, borders
  - `#F1F5F9` (Slate-100) - Backgrounds
  - `#FFFFFF` - Pure white

#### Gradient Options
- **Hero Gradient:** `from-blue-600 via-blue-500 to-orange-500`
- **Card Gradient:** `from-blue-50 to-orange-50`
- **Button Gradient:** `from-blue-600 to-blue-700`

**Rationale:** Blue conveys trust and reliability (essential for research tools), while keeping orange as accent maintains energy and differentiation. The color scheme emphasizes professionalism and accuracy over flashy tech aesthetics.

### 1.3 Typography

**Current:** Geist Sans, Geist Mono
**Recommended Hierarchy:**

#### Headings
- **H1 (Hero):** Geist Sans, 48-72px, Weight 800, Line Height 1.1
- **H2 (Section):** Geist Sans, 36-48px, Weight 700, Line Height 1.2
- **H3 (Subsection):** Geist Sans, 24-30px, Weight 600, Line Height 1.3
- **H4 (Card Title):** Geist Sans, 20px, Weight 600, Line Height 1.4

#### Body Text
- **Large:** Geist Sans, 18px, Weight 400, Line Height 1.6
- **Base:** Geist Sans, 16px, Weight 400, Line Height 1.6
- **Small:** Geist Sans, 14px, Weight 400, Line Height 1.5

#### Code/Mono
- **Code Blocks:** Geist Mono, 14px, Weight 400
- **Inline Code:** Geist Mono, 14px, Weight 500

### 1.4 Logo Design Concepts

**Option 1: Modern Wordmark**
- "QueryAI" in Geist Sans Bold
- "Q" letterform with subtle search/research icon (magnifying glass or document)
- Blue gradient (#2563EB to #3B82F6)
- Icon: Stylized "Q" with search/document symbol

**Option 2: Icon + Wordmark**
- Circular icon: Magnifying glass + document/checkmark (research/verification)
- Colors: Blue gradient with orange accent
- Wordmark: Clean, modern sans-serif
- Horizontal and vertical lockups

**Option 3: Minimalist**
- Simple "Q" monogram with research/search element
- Geometric, professional
- Works at small sizes
- Full wordmark for larger applications

**Recommendation:** Start with Option 2 (icon + wordmark) emphasizing research/verification over AI/tech imagery.

### 1.5 Visual Elements

**Icons:**
- Style: Line icons with 2px stroke
- Library: Lucide React (already in use) or custom
- Consistency: Rounded corners, consistent stroke weight

**Illustrations:**
- Style: Modern, minimalist, tech-forward
- Use for: Empty states, feature explanations
- Color: Blue/orange gradient fills

**Patterns/Textures:**
- Subtle grid patterns for backgrounds

**Animations:**

## üé® 4. UI/UX DESIGN SYSTEM

### 4.1 Design Principles

1. **Clarity First** - Clear hierarchy, readable text
2. **Consistency** - Unified components, patterns
3. **Accessibility** - WCAG 2.1 AA compliance
4. **Performance** - Fast, responsive interactions
5. **Delight** - Subtle animations, micro-interactions

### 4.2 Component Library Updates

#### Buttons
**Variants:**
- **Primary:** Blue gradient, white text, shadow
- **Secondary:** Outline, blue border, blue text
- **Ghost:** Transparent, blue text on hover
- **Danger:** Red background for destructive actions
- **Success:** Green for positive actions

**States:**
- Default, Hover, Active, Disabled, Loading
- Size variants: sm, md, lg

#### Cards
**Styles:**
- **Elevated:** White background, shadow, rounded-lg
- **Outlined:** Border, no shadow
- **Gradient:** Subtle gradient background
- **Interactive:** Hover lift effect, cursor pointer

#### Input Fields
**Design:**
- Rounded corners (8px)
- Clear focus states (blue ring)
- Error states (red border, message)
- Success states (green border)
- Helper text support

#### Navigation
**Header:**
- Sticky on scroll
- Logo left, nav center, CTA right
- Mobile: Hamburger menu
- Active state indicators

**Sidebar (Dashboard):**
- Collapsible
- Icon + text labels
- Active state highlighting
- Badge support for counts

#### Chat Interface
**Improvements:**
- Message bubbles with rounded corners
- User messages: Blue, right-aligned
- AI messages: Gray/white, left-aligned
- Source citations: Expandable, styled
- Typing indicator: Animated dots
- Timestamp: Subtle, small text

#### Forms
**Design:**
- Clear labels above inputs
- Inline validation
- Error messages below fields
- Success confirmation
- Loading states during submission

### 4.3 Dashboard Redesign

#### Layout
**Structure:**
- Left sidebar: Navigation (collapsible)
- Main content: Tabbed interface
- Right panel: Contextual info (optional)

#### Tab Navigation
**Design:**
- Horizontal tabs at top
- Active tab: Blue underline, bold text
- Icons + labels
- Badge for notifications/counts

#### Chat Tab
**Improvements:**
- Cleaner message layout
- Better source citation display
- Improved input area
- Quick action buttons
- Conversation history sidebar

#### Document Manager
**Improvements:**
- Grid/list view toggle
- Drag-and-drop upload
- Progress indicators
- File type icons
- Quick actions menu

#### Topic Manager
**Improvements:**
- Visual topic cards
- Color coding
- Quick stats per topic
- Edit/delete actions
- Search/filter

### 4.4 Dark Mode Support

**Color Scheme:**
- Background: `#0F172A` (Slate-900)
- Surface: `#1E293B` (Slate-800)
- Text: `#F1F5F9` (Slate-100)
- Accent: Blue-500, Orange-500
- Borders: Slate-700

**Implementation:**
- System preference detection
- Manual toggle in settings
- Smooth theme transition
- Persist user preference

### 4.5 Accessibility Improvements

**WCAG 2.1 AA Compliance:**
- Color contrast ratios (4.5:1 minimum)
- Keyboard navigation support
- Screen reader labels
- Focus indicators
- Alt text for images
- ARIA labels where needed

**Testing:**
- Keyboard-only navigation
- Screen reader testing (NVDA/JAWS)
- Color blindness simulation
- Zoom testing (200%)

### 4.6 Animation & Micro-interactions

**Principles:**
- Subtle, purposeful animations
- 200-300ms duration
- Easing: ease-out for most
- Reduce motion for accessibility

**Animations:**
- Button hover: Scale 1.02, shadow increase
- Card hover: Translate Y -4px, shadow increase
- Page transitions: Fade in (300ms)
- Loading: Smooth spinner, skeleton screens
- Success: Checkmark animation
- Error: Shake animation (subtle)

## üìù 9. CONTENT GUIDELINES

### Tone of Voice
- **Professional yet approachable**
- **Clear and concise**
- **Confident but not arrogant**
- **Helpful and educational**

### Writing Style
- Use active voice
- Short sentences (15-20 words)
- Bullet points for lists
- Headlines: 6-8 words
- Subheadlines: 10-15 words

### Terminology
- **Primary Focus:** Research, facts, verification, sources
- **Technology Mention:** Mention AI/technology only when necessary, focus on outcomes
- Use "research assistant" or "fact research" over "AI tool"
- "Find facts" or "research questions" not "ask AI"
- "Source-cited answers" not "AI-generated responses"
- "Verified information" not "AI answers"
- "Research tool" not "AI platform"
- Keep technology mentions subtle: "powered by advanced technology" rather than "AI-powered"
